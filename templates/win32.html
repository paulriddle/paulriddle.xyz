Notes

Even though it is called Win32 the layer will be written with 64-bit architecture in mind. Win32 is just the name the API arrived at historically. Microsoft decided that renaming Win32 to Win64 doesn't make much sense because Win32 can be easily compiled for both 32-bit and 64-bit. But compiling Win64 in 32-bit mode might be confusing.

Win32 is not deprecated. For Microsoft, deprecation of an API is more of a marketing statement rather than commitment to removing the API in the future.

WinMain vs wWinMain

From the docs:

The WinMain function is identical to wWinMain, except the command-line arguments are passed as an ANSI string. The Unicode version is preferred. You can use the ANSI WinMain function even if you compile your program as Unicode.
